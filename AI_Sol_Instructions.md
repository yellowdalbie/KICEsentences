# KICE 수학 문항 해설 작성 지시서 (AI 실행 명령서)

이 문서는 AI가 새 세션에서 해설 작성 작업을 시작할 때 반드시 먼저 읽어야 하는 실행 지시서입니다.
`AI_Instruction_KICE_System.md`(프로젝트 목적/아키텍처)와 `AI_Template_Commentary.md`(기본 템플릿)를 **상위 문서**로 두고, 이 문서는 실제 작업 과정에서 지켜야 할 세부 규칙을 보완합니다.

---

## 0. 작업 시작 전 준비

1. `MD_Ref/{년도}/{파일명}.md`를 반드시 먼저 읽는다.
2. 문항 전체를 머릿속에서 풀고, 논리적 도약이 발생하는 지점마다 Step을 설정한다.
3. 저장 위치: `Sol/{년도}/{파일명}.md`

---

## 1. 파일 헤더 형식

```markdown
# {파일명} 해설
>
> 원본 문항 링크: [{파일명}](../../MD_Ref/{년도}/{파일명}.md)
```

예시:

```markdown
# 2026.9모_12.md 해설
>
> 원본 문항 링크: [2026.9모_12.md](../../MD_Ref/2026/2026.9모_12.md)
```

---

## 2. Step 블록 형식

각 Step은 아래 4개 블록을 순서대로 작성한다. 블록 사이에 빈 줄을 둔다.

```markdown
## [Step N] {타이틀}

- **Trigger**: [{카테고리}] {조건 수식 또는 서술} ({괄호 부가설명})
- **Action**: [{CPT 코드}] {구체적 수학적 조작}
- **Result**: {결과 수식 또는 서술}

> **📝 해설 (Explanation)**
> {친절한 해설 줄글}
```

---

## 3. Step 타이틀 작성 규칙

**형식**: `[개념/방법] + [대상/결과]`

| 금지 (❌) | 허용 (✅) |
|-----------|-----------|
| `$f(1)$ 계산` | `초기 조건으로 구한 $f(x)$에 $x=1$ 대입` |
| `$a_2$ 계산` | `공차 적용으로 $a_2$ 산출` |
| `합산` | `등비수열 합 공식으로 부분합 산출` |
| `확률 계산` | `경우의 수 비율로 확률 산출` |
| `최종 연산` | `분리된 시그마 합으로 최종값 산출` |

- **수식+단순행위어** 타이틀 금지: 수식이 무슨 역할인지 개념어가 없는 것
- **추상적 행위어** 타이틀 금지: 방법/개념 없이 행위만 있는 것
- 벡터 유사도 기반 시스템이므로 표현의 완전한 표준화는 불필요. 단, 의미 정보 자체가 없는 타이틀은 반드시 수정한다.

---

## 4. Trigger 작성 규칙

**혼합형 필수**: `[카테고리명]` + 구체적 수식/서술 + `(괄호 부가설명)`

```
[삼각함수의 각변환 조건] cos(π/2+θ) = √5/5, tanθ < 0 (각변환 후 사분면 결정)
```

- 카테고리만 있으면 안 됨
- 수식만 있어도 안 됨
- 둘 다 있어야 함

### 자주 쓰이는 Trigger 카테고리

| 카테고리명 | 사용 맥락 |
|-----------|----------|
| `[도함수의 식 제시]` | $f'(x)=\cdots$과 초기값이 주어진 경우 |
| `[등차수열의 조건식 제시]` | 등차수열 조건 2개 이상 |
| `[등비수열의 조건식 제시]` | 등비수열 조건 2개 이상 |
| `[삼각함수의 각변환 조건]` | $\cos(\pi/2+\theta)$ 등 각변환 주어짐 |
| `[삼각함수 부호 조건]` | 사분면 정보로 삼각함수 부호 결정 |
| `[다항함수의 극값 조건]` | 극대/극소 $x$ 좌표 또는 함숫값 조건 |
| `[이차방정식의 해와 조건을 이용한 공비 확정]` | 방정식의 해 중 조건에 맞는 것 선택 |
| `[배반사건 조건]` | $A \cap B = \emptyset$ 또는 이와 동치인 조건 |
| `[여사건 확률 조건]` | $P(A^C) = 1 - P(A)$ 적용 |
| `[확률 덧셈 조건]` | 덧셈정리 또는 분해 공식 적용 |
| `[확률 대수 조건]` | $P(A \cup B)$, $P(A \cap B)$ 등 여러 확률값 제시 |
| `[이산확률변수 조건]` | 확률분포표 또는 확률변수 정의 |
| `[이산확률변수 분산 조건]` | $V(X) = E(X^2) - [E(X)]^2$ 적용 |
| `[시그마 기호식 조건]` | $\sum$ 형태의 조건식이 제시됨 |
| `[두 점 사이 거리 조건]` | $\overline{AB} = \overline{AC}$ 등 거리 등치 조건 |
| `[삼각형 넓이 조건]` | 넓이가 특정 값으로 주어짐 |
| `[두 로그 방정식 제시]` | 로그 방정식 2개 이상이 주어짐 |
| `[지수함수 조건]` | $y = a^x$ 또는 $y = a^x + k$ 등 지수함수 그래프 조건 |
| `[지수함수 평행이동 조건]` | $y = a^x + k$ 꼴의 평행이동된 지수함수 |
| `[기댓값 계산 조건]` | 결정된 파라미터로 $E(X)$, $E(X^2)$ 계산 |
| `[최종 구하는 값]` | 마지막 Step에서 문제가 구하라 한 값 |

## 5. Action 작성 규칙

- `concepts.json`의 유효한 CPT ID를 반드시 앞에 명기
- **"및", "또는"이 들어가 두 조작을 설명하면 안 됨** → Step 분리 대상
- 형식: `[CPT-XXX-YYY-NNN] {구체적 수학적 조작}`

### 자주 쓰이는 CPT 코드

| CPT 코드 | 내용 |
|----------|------|
| `CPT-ALG-SEQ-001` | 등차수열 일반항 |
| `CPT-ALG-SEQ-002` | 등비수열 일반항 |
| `CPT-ALG-SEQ-003` | 등비수열 합 공식 |
| `CPT-ALG-SEQ-004` | 시그마($\sum$) 성질/계산 |
| `CPT-ALG-TRI-002` | 삼각함수 항등식 및 각변환 |
| `CPT-ALG-EXP-001` | 지수함수 |
| `CPT-ALG-EXP-004` | 로그 및 로그함수 |
| `CPT-CA1-DIF-003` | 다항함수 미분 및 극값 |
| `CPT-CA1-DIF-007` | 극값의 정의와 활용 |
| `CPT-CA1-INT-002` | 부정적분과 초기조건으로 함수 결정 |
| `CPT-STA-PRO-002` | 확률의 덧셈정리 및 분해 |
| `CPT-STA-PRO-003` | 여사건의 확률 |
| `CPT-STA-STA-001` | 이산확률변수의 확률분포 |
| `CPT-STA-STA-002` | 기댓값, 분산, 표준편차 |

> 위 목록에 없는 새 CPT ID가 필요하다고 판단되면 **독단 작성 금지**. 사용자에게 먼저 질의한다.

---

## 6. Result 작성 규칙

- 수식 결과 + 수학적 의미 또는 다음 단계를 위한 해석 병기
- 예: `$r=2$ (모든 항 양수이므로 양의 공비 확정)`
- 예: `$\sin\theta = -\dfrac{\sqrt{3}}{3}$ (4사분면에서 음수임을 확인)`
- 마지막 Step의 Result: 정답값만 간결하게 표기

---

## 7. 해설(Explanation) 작성 규칙

- 독립적으로 학생에게 보여진다는 것을 가정하고 작성
- 연산 과정 생략 없이 전부 표시
- 해당 Action을 선택한 필연성(왜 이 방법인지) 설명 권장
- 종결어미: **`~입니다` 체** 일관 유지 (`~이다` 금지)
- **문체 (Tone & Manner)**: 철저히 수학적이고 건조한 학술적 문어체를 사용. 비유적 표현이나 구어체("땀 흘려", "친절하게", "관문", "덩어리", "깎아나가며", "외톨이", "진짜로" 등) 절대 사용 금지.
- **교육과정 준수**: 2022 개정 교육과정 기준에 부합하는 수학 용어 사용 (예: 확률과 통계에서 교육과정 외인 '자연수의 분할' 용어 및 개념 접근 지양 $\rightarrow$ '제약 조건에 따른 체계적인 분류' 및 '합의 법칙' 또는 '곱의 법칙'으로 논리 전개).
- **마지막 Step의 마지막 줄**: 정답 명시
  - 선택지형: `정답은 **③**입니다.` 또는 `따라서 정답은 **③**입니다.`
  - 단답형: `정답은 **15**입니다.`

---

## 8. Step 분리 기준

### 분리 필수

- 서로 다른 수학적 판단이 2개 이상 들어갈 때
  - 조건으로 상수/공비 결정 (Step N) / 결정된 값으로 목표값 계산 (Step N+1)
  - 사분면으로 부호 판단 (Step N) / 항등식으로 크기 계산 (Step N+1)
  - 방정식 수립 (Step N) / 방정식 풀기 (Step N+1) — 단, 단순한 경우 통합 허용

### 통합 허용

- 연산의 흐름상 논리적 단절이 굳이 없는 경우
  - 이차방정식 인수분해 후 두 근 동시 도출
  - 좌극한·우극한 계산 후 연속 조건 수립

### 단순 문항

- 1~2 Step으로 끝나도 반드시 `[Step 1]` 형식 유지

---

## 9. 새 패턴 발견 시 처리 원칙

기존 Trigger 카테고리나 CPT 코드로 표현이 어려운 새 패턴 발견 시:

1. 독단으로 작성하지 않는다
2. 사용자에게 "이런 조건/반응이 나왔는데, 통제어 사전에 어떻게 편입할까요?"라고 먼저 질의한다

## 10. 작성 예시 (참조용)

```markdown
# 2023.9모_05.md 해설
>
> 원본 문항 링크: [2023.9모_05.md](../../MD_Ref/2023/2023.9모_05.md)

## [Step 1] 등차수열 조건으로 연립방정식 수립

- **Trigger**: [등차수열의 조건식 제시] $a_1=2a_5$, $a_8+a_{12}=-6$ (두 등차 조건)
- **Action**: [CPT-ALG-SEQ-001] $a_n = a_1 + (n-1)d$로 표현: $a_1=2(a_1+4d) \Rightarrow -a_1=8d$, $2a_1+18d=-6$
- **Result**: $-a_1=8d$에서 $a_1=-8d$. 대입: $-16d+18d=2d=-6 \Rightarrow d=-3$, $a_1=24$

> **📝 해설 (Explanation)**
> $a_1=2a_5$: $a_1=2(a_1+4d) \Rightarrow -a_1=8d$
> $a_8+a_{12}=(a_1+7d)+(a_1+11d)=2a_1+18d=-6$
> $a_1=-8d$ 대입: $2d=-6 \Rightarrow d=-3$, $a_1=24$

## [Step 2] 공차 적용으로 $a_2$ 산출

- **Trigger**: [최종 구하는 값] $a_2$
- **Action**: [CPT-ALG-SEQ-001] $a_2 = a_1 + d = 24 + (-3) = 21$
- **Result**: $a_2 = 21$

> **📝 해설 (Explanation)**
> $a_2 = 24 + (-3) = 21$이며, 정답은 **③**입니다.
```
