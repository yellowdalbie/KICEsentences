## [Step 1] 로그 지표의 범위 분류로 조건 충족 경우 탐색

- **Trigger**: [로그 지표 조건] $f(ab) = f(a)f(b) + 2$를 만족하는 20 이하 자연수 $a, b$ 탐색
- **Action**: [12대수01-04] 상용로그의 지표 $f(x) = \lfloor \log x \rfloor$는 $1 \leq x \leq 9$이면 $0$, $10 \leq x \leq 99$이면 $1$임을 이용하여 $f(a), f(b)$의 가능한 조합을 경우별로 분류
- **Result**: $f(a) = 0, f(b) = 1$ (또는 반대)이고 $ab \geq 100$인 경우만 조건 충족

> **📝 해설 (Explanation)**
> 20 이하 자연수에서 지표값: $a \in \{1, \ldots, 9\}$이면 $f(a) = 0$, $a \in \{10, \ldots, 20\}$이면 $f(a) = 1$.
>
> 경우 1: $f(a)=0, f(b)=0$ → $f(ab) = 0 \cdot 0 + 2 = 2$이므로 $ab \geq 100$이어야 함.
> $a \leq 9, b \leq 9$이면 $ab \leq 81 < 100$. 불가능.
>
> 경우 2: $f(a)=0, f(b)=1$ → $f(ab) = 0 \cdot 1 + 2 = 2$이므로 $100 \leq ab \leq 999$이어야 함.
> $a \in \{1, \ldots, 9\}, b \in \{10, \ldots, 20\}$이면 $ab \leq 9 \times 20 = 180 < 1000$이므로 상한은 자동 만족.
> 조건: $ab \geq 100$.
>
> 경우 3: $f(a)=1, f(b)=1$ → $f(ab) = 1 \cdot 1 + 2 = 3$이므로 $ab \geq 1000$이어야 함.
> $a, b \leq 20$이면 $ab \leq 400 < 1000$. 불가능.
>
> 따라서 경우 2(또는 대칭인 $f(a)=1, f(b)=0$)만 가능합니다.

## [Step 2] 조건 충족 순서쌍 탐색으로 a+b 최솟값 결정

- **Trigger**: [최적화 조건] $a+b$의 최솟값 요구
- **Action**: [10공수1-02-09] $a \in \{1,...,9\}$, $b \in \{10,...,20\}$에서 $ab \geq 100$을 만족하는 순서쌍 중 $a+b$가 최소인 것을 탐색
- **Result**: $(a,b) = (8,13)$ 또는 $(9,12)$에서 $a+b = 21$이 최솟값

> **📝 해설 (Explanation)**
> $a + b$가 작으려면 $a$가 크고 $b$가 작아야 합니다.
>
> $a + b = 20$인 조합 검토 ($a \leq 9, b \leq 20$에서 $a+b=20$):
> - $(1,19): ab=19$, $(2,18): ab=36$, $(3,17): ab=51$, $(4,16): ab=64$
> - $(5,15): ab=75$, $(6,14): ab=84$, $(7,13): ab=91$, $(8,12): ab=96$, $(9,11): ab=99$
> 모두 $ab < 100$이므로 불가능.
>
> $a + b = 21$인 조합 검토:
> - $(8,13): ab = 104 \geq 100$ ✓, $f(104) = 2 = f(8) \cdot f(13) + 2 = 0 \cdot 1 + 2$ ✓
> - $(9,12): ab = 108 \geq 100$ ✓, $f(108) = 2 = f(9) \cdot f(12) + 2 = 0 \cdot 1 + 2$ ✓
>
> $a + b$의 최솟값은 $\mathbf{21}$입니다.
> 따라서 정답은 **③**입니다.
