# Dialectical Memory (누적 출제/검토 매뉴얼)

이 문서는 수학 문항 해설 작성 시 발생한 오류, 금칙어, 교육과정 경계 예외 케이스들을 누적 저장하는 단일 지식 베이스입니다. 에이전트(출제위원장)는 파일 전체 파편화를 방지하기 위해 해설 작성 전 이 문서를 `grep_search`하여 필요한 주의사항을 핀포인트로 스캔합니다.

## [공통 지침] 수식 표기 규정
- **타이틀 수식 금지**: 타이틀은 벡터 임베딩을 위해 정형화된 형태를 유지해야 하므로 구체적 수식 포함 절대 금지.
- **Action 수식 권장**: Action 블록에는 오히려 구체적인 수식이나 조작식(예: `$8=2^3$ 대입`)을 적극적으로 기입하여, 해당 문항만의 개별성을 보존할 것.
- **정답 문구 표준화**: 해설의 마지막 줄은 반드시 **`따라서 정답은 **N**입니다.`** (번호 문항) 또는 **`따라서 정답은 **N**입니다.`** (단답형) 형식을 준수하여 논리적 매듭을 지을 것. **'따라서'라는 단어는 오직 이 마지막 줄에서만 정답 시그널로 사용하며, 해설 중간 단계의 인과관계 표현에는 '그러므로', '이를 통해', '결과적으로' 등 다른 연결어를 사용하여 엄격히 구분할 것.**
- **타이틀 내 단순 산술 표현 지양**: "의 합 산출", "의 차 계산" 등 본질적 수학 개념 뒤에 붙는 단순 산술 연산 표현은 벡터 군집화를 위해 지양하고, 반드시 프로젝트 표준 용어인 **"구하기"**로 종결할 것. (예: "확정하기", "산출하기", "변환하기" 대신 "구하기" 사용)
- **용어 제어 (금지어)**: '단조 증가/감소'(증가, 감소로 순화), '확률 모수'(확률로 순화), '계차수열'(수열의 귀납적 정의 등으로 풀어서 서술), '점화식'(수열의 귀납적 정의로 순화) 등 교육과정 외 용어 절대 사용 금지.

- **스텝 분리 기준**: 단순히 수치 대입이나 표준적인 표 읽기 등 '단순 계산'에 가까운 과정은 별도 스텝으로 분리하지 않고 핵심 개념과 병합함. 단, 정규분포의 대칭성 활용이나 특수한 발상이 필요한 단계는 해당 아이디어를 타이틀에 명시하여 독립 스텝으로 분리할 수 있음.

- **스텝 타이틀의 보편성 확보**: 타이틀 작성 시 '홀수', '짝수', '상자', '공' 등 문항 특수적인 소재를 직접 노출하기보다 **'시행 결과의 성질'**, **'사건의 발생 횟수'**, **'특정 조건을 만족할 확률'** 등 수학적/통계학적 보편 용어를 사용하여 추상도를 높일 것. 단, 해설 본문(`📝 해설`)에서는 학습자의 이해를 위해 문항의 소재를 적극 활용함.
- **대학 수준 용어 사용 금지**: **'합동식'**, **'법(Modulo)'**, **'가중치(Weight)'** 등 고교 교육과정을 벗어난 대학 통계학/정수론 용어는 타이틀 및 해설 전반에서 절대 사용 금지.
- **문항 특수 소재의 추상화**: 타이틀 및 핵심 Action 작성 시 '상자', '공', '주머니' 등 문항의 소재에 의존하는 표현을 지양하고, 이를 **'누적 발생 횟수'**, **'가중치(Weight)'**, **'원소의 집합'** 등 수학적 본질을 담은 용어로 치환하여 기술할 것. 이는 벡터 임베딩의 일반성과 검색 변별력을 높이기 위함임.

### 🛡️ 에이전트 행동 철칙 (Ironclad Rules)
- **제로베이스 리스캔(Zero-based Rescan)**: 매 문항의 진단 및 검토 시마다 `MD_Ref`, `Sol`, `Docs` 내 모든 매뉴얼 파일을 새로 읽어(view_file) 이전 문항의 잔상이나 잘못된 관습이 개입되지 않도록 함. (평가원장 엄중 지시 사항)
- **고밀도 스텝 통합**: 4점 고난도 문항이라 하더라도 동일한 수학적 알고리즘(예: 중복조합 모델링)이 반복 적용되는 경우, 개별 케이스의 산출 과정을 별도 스텝으로 분절하기보다 **'단일 모델의 다각적 적용'** 관점에서 하나의 스텝으로 통합하여 논리적 선명도를 높일 것.
